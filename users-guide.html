1. 介绍

1.1. 软件用途

openTCS（open Transportation Control System 的缩写）是一款免费的控制系统软件，用于协调自动导引车（AGV）和移动机器人车队，例如在生产工厂中。open Transportation Control System）是一款用于协调车队自动导引车（AGVs）和移动机器人的免费控制系统软件，例如在生产工厂中。

openTCS 控制车辆时独立于其具体特性，如导航原理/轨道引导系统或负载处理装置。

openTCS 本身并不是一个开箱即用以控制 AGV 的完整产品。

因此，通常至少需要创建并插入一个车辆驱动程序（在 openTCS 术语中称为通信适配器），该驱动程序负责在 openTCS 内核的抽象接口与您的车辆所理解的通信协议之间进行转换。通信适配器（在 openTCS 术语中）负责在 openTCS 内核的抽象接口与您车辆所理解的通信协议之间进行转换。

2.1.4. 使用图层和图层组

除了向工厂模型添加元素外，模型编辑器还允许使用不同的图层和图层组创建工厂模型。Layer and 图层组.

在操作台应用程序中，只能显示/隐藏图层和图层组，但不能对其进行操作。
2.1.4.1. 图层

图层对点位、路径、位置和链接进行分组，并允许按需显示或隐藏其包含的行驶路线元素。

  • 必须至少存在一个图层。

  • 添加新图层时，它始终成为活动图层。

  • 删除图层会导致其包含的行驶路线元素也被删除。

  • 当添加模型元素（即点位、路径等）时，它们始终放置在活动图层上。

  • 链接（位置和点位之间）始终放置在与相应位置相同的图层上，无论当前活动图层是什么。

  • 对行驶路线元素执行“复制并粘贴”、“剪切并粘贴”或“重复”操作时，副本始终放置在活动图层上。

在操作台应用程序中，图层（和图层组）的可见性也会影响车辆元素是否在工厂模型中显示。

2.1.4.2. 图层组

正如名称所示，图层组对一个或多个图层进行分组，并允许同时显示或隐藏多个图层。

  • 必须至少存在一个图层组。

  • 可以通过在外设作业表中选择它们并单击上面显示的撤回按钮来撤回与运输单无关的外设作业以及与处于最终状态（已完成或失败）的运输单相关的外设作业。

  • 当相应的运输单被撤回时，与运输单相关的外设作业将隐式中止。

    • 对于常规撤回的运输单，相关的外设作业将在车辆完成其待处理的移动后中止。

    • 对于强制撤回的运输单，相关的外设作业将立即中止。

2.2.8.1. 外设任务失败的后果

当外设作业失败时，相关的运输单（如果有）会在以下情况下被隐式撤回：

  • 该外设作业是由车辆遍历路径隐式创建的（参见外设作业的隐式创建).

  • The 需要完成外设作业的“需要完成”标志设置为true.

在这种情况下，继续运输单是不合理的，因为“完成必需”标志不仅表示车辆应等待外设操作完成，还表示必须成功完成。需要完成标志不仅指示车辆应等待外设操作完成，还指示其必须完成successfully. Otherwise, the prerequisites for continuing driving / the transport order may not be met.

4.6.2. 路径

路径是点位之间供车辆通行的连接。

  • Its length，这在工厂运行模式下可能是与车辆相关的信息。

  • A 最大速度 and 最大反向速度，这在工厂运行模式下可能是与车辆相关的信息。

  • A locked标志，当设置时，告知路由器该路径在计算车辆路线时不可使用。

  • A sequence of 外设操作描述当车辆遍历路径时由外设设备（按给定顺序）执行的操作。

  • A set of 车辆包络线描述遍历路径的车辆所占用的区域。

4.6.2.1. 外设操作

外设操作的属性包括：

  • A reference to the location的引用，表示将通过该外设执行操作 — 参见Location.

  • 实际operation由外设执行。

  • An 执行触发器定义执行操作的时刻。

    • AFTER_ALLOCATION：在车辆分配路径后触发操作执行。

    • AFTER_MOVEMENT：在车辆遍历路径后触发操作执行。

  • A 完成必需标志，当设置时，要求完成操作以允许车辆继续行驶。AFTER_ALLOCATION执行触发器和完成必需标志设置为true时，车辆必须在路径的源点等待，直到操作完成。AFTER_MOVEMENT执行触发器和完成必需标志设置为true时，车辆必须在路径的目标点等待，直到操作完成。

4.6.3. 位置

位置是用于标记车辆可执行特殊操作（如装载或卸载货物、充电等）的点位。

  • Its type，基本定义允许在该位置执行哪些操作 — 参见位置类型.

  • A set of links到可以从该位置到达的点位。

  • A locked标志，当设置时，告诉调度器可能不会将需要在该位置执行操作的运输单分配给车辆。

此外，位置还可以映射外设设备，以便与其通信并允许车辆与之交互（例如，沿路径打开/关闭防火门）。添加和配置外设有关如何添加和配置外设的详细信息。

4.6.4. 位置类型

位置类型是对位置进行分组的抽象元素。

  • A set of 允许/支持的车辆操作，定义车辆可以在这种类型的位置执行哪些操作。

  • A set of 允许/支持的外设操作，定义映射到这种类型位置的外设设备可以执行哪些操作。

4.6.5. 车辆

车辆映射物理车辆，以便与其通信并可视化其位置和其他特征。

  • 一组能量等级阈值，其组成如下：

    • A 临界能量等级，低于该阈值时，车辆的能量等级被视为临界。

4.9.3.1. 分配资源

  1. 检查请求资源的车辆是否not paused.

  2. 检查请求的资源是否对车辆一般可用。

  3. 检查请求的资源是否属于类型为SINGLE_VEHICLE_ONLY. If not, skip this check. If yes, expand the requested resource set to the effective resource set and check if the expanded resources are available for the vehicle.

  5. 检查与请求资源相关的区域是否可供该车辆使用且未被其他车辆分配（前提是请求资源的车辆引用了包络线键，且请求的资源定义了具有该键的车辆包络线）。

      • 它的预留令牌必须未设置。

      • 它必须可供外围设备处理。

      • 它必须能够被外围设备处理。
